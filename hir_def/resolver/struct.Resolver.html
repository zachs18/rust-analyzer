Note that this is a somewhat fuzzy concept â€“ internally, the compiler doesnâ€™t necessary follow a strict scoping discipline. Rather, it just tells for each ident what it resolves to.

A good example is something like str::from_utf8. From scopes point of view, this code is erroneous â€“ both str module and str type occupy the same type namespace.

We donâ€™t try to model that super-correctly â€“ this functionality is primarily exposed for completions.

Note that in Rust one name can be bound to several items:

macro_rules! t { () => (()) }
type t = t!();
const t: t = t!()

Thatâ€™s why we return a multimap.

The shadowing is accounted for: in

{
    let x = 92;

}

there will be only one entry for x in the result.

The result is ordered roughly from the innermost scope to the outermost: when the name is introduced in two namespaces in two scopes, we use the position of the first scope.
